# Domain: Persistence for Non-Technical Teams

## Core Concept

Accessibility over complexity. Non-technical stakeholders shouldn't need Git.

Google Drive is familiar. We extend it with folder conventions, sync logic, and role-based access.

## Principles

1. **Folder as Database** — Drive structure replaces Git branches
2. **Timestamp as Version** — Files versioned by modification time
3. **Push/Pull, not Commit** — Familiar folder sync metaphor (Dropbox)
4. **Role-Based Access** — Folder-level permissions, not per-file
5. **Local-First Conflicts** — User changes always win
6. **Gradual Adoption** — Works alongside Git (both valid)

## Clara Values

- **Clarity** — Structured folders, clear permissions, no Git jargon
- **Autonomy** — Users control their project context
- **Efficiency** — One-command sync vs. complex Git workflow
- **Trust** — Role-based access prevents accidents
