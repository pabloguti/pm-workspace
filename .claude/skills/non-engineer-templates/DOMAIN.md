# Non-Engineer Templates Domain

## Overview

This domain enables guided, non-technical interaction with PM-Workspace across three key personas: Product Owners, Stakeholders, and QA teams.

## Scope

- **Who:** POs, stakeholders, QA engineers without PM-Workspace technical training
- **What:** Simplified wizards, executive dashboards, structured forms
- **How:** Natural language, step-by-step guidance, plain English prompts
- **Why:** Democratize PM-Workspace access, reduce dependency on technical intermediaries

## Key Capabilities

- Sprint planning and backlog management for POs
- Executive reporting and milestone tracking for stakeholders
- Bug reporting and test planning for QA teams

## Design Principles

1. **No Technical Jargon** - Use plain language throughout
2. **Guided Workflows** - Step-by-step, linear interaction flows
3. **Role-Specific Views** - Each persona sees only relevant information
4. **Minimal Cognitive Load** - Simplify complexity, hide unnecessary details
