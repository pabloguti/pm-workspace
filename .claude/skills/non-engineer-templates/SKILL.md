# Non-Engineer Templates

**Name:** non-engineer-templates

**Description:** Guided templates designed for Product Owners, stakeholders, and QA to interact with PM-Workspace without requiring technical knowledge. Provides simplified command sets, guided wizards, and natural language interfaces tailored to each persona.

## Personas

### Product Owner (PO)
- **Role:** Manages product backlog, establishes priorities, defines acceptance criteria
- **Interaction Style:** Simplified command interface, sprint planning wizards, backlog prioritization workflows
- **Key Actions:** Plan sprints, prioritize features, manage acceptance criteria, review completed work

### Stakeholder
- **Role:** Views reports, provides feedback, reviews project milestones
- **Interaction Style:** Executive-level dashboard, natural language reporting, milestone review workflows
- **Key Actions:** View executive summaries, track milestones, assess risks, review budget impact

### QA / Quality Assurance
- **Role:** Defines test plans, reports bugs, validates acceptance criteria
- **Interaction Style:** Structured bug report wizards, test plan creation workflows, regression testing interfaces
- **Key Actions:** Create test plans from PBIs, report bugs systematically, validate acceptance criteria, plan regression testing

## Templates & Workflows

### PO Sprint Planning Wizard
Step-by-step guided flow for planning upcoming sprints. Assists in selecting backlog items, estimating capacity, and setting sprint goals without technical jargon.

### PO Backlog Prioritization
Interactive workflow for re-prioritizing backlog items based on business value, dependencies, and stakeholder feedback. Uses natural language sorting and prioritization.

### Stakeholder Executive Summary Request
On-demand dashboard view showing high-level project status, key milestones, risks, budget status, and progress metrics in executive-friendly language.

### Stakeholder Milestone Review
Guided review of completed project milestones, deliverables achieved, timeline adherence, and stakeholder sign-off workflow.

### QA Test Plan from PBI
Automated workflow that transforms Product Backlog Items into structured test plans with test scenarios, acceptance criteria validation, and coverage mapping.

### QA Bug Report Wizard
Structured wizard for reporting bugs with categorization, severity assessment, reproduction steps, environment details, and impact analysis without technical setup required.

## Core Principle

All templates follow a **step-by-step guided flow** using plain language, avoiding technical jargon, ensuring non-technical team members can independently manage their domain responsibilities within PM-Workspace.
