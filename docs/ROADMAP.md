## Evaluation Criteria

Every backlog item is scored (1–5) on 5 dimensions:

| Dimension | Weight | Description |
|-----------|--------|-------------|
| **PM Impact** | 30% | Does it improve real project management? |
| **Anti lock-in** | 25% | Works offline/standalone? No vendor dependency? |
| **FOSS** | 20% | Uses/generates free software? Contributes to ecosystem? |
| **Inverse complexity** | 15% | 5=trivial, 1=months of work |
| **Savia Flow alignment** | 10% | Fits the 6 principles: team choice, outcomes, quality, humans in center, progressive, data over opinions |

FOSS alternatives chosen for every layer:

| Proprietary | FOSS preferred |
|------------|----------------|
| FCM (Google) | WebSocket + ntfy.sh |
| Pinecone/Weaviate | sqlite-vec + sentence-transformers |
| Okta SDK | Keycloak + OIDC standard |
| New Relic/Datadog | OpenTelemetry |
| Zapier/Make | n8n / Automatisch / Activepieces |
| Tableau/Power BI | Apache Superset / Metabase |

## How to influence the roadmap

1. Check if your idea already has an open issue — if so, add a reaction.
2. If not, open a new issue using the **Feature request** template.
3. The most-voted open issues are pulled into the next milestone during planning.
4. Want to implement something? Comment on the issue first — maintainers will confirm the approach fits before you invest time in a PR.
