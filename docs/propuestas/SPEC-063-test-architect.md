# SPEC-063 — Test Architect Agent

**Status:** Draft | **Author:** Savia | **Date:** 2026-03-29

## Problem

pm-workspace generates tests across 16 language packs and 14 test types, but
test quality varies wildly. Session 2026-03-30 showed it took 3 rounds of
iteration to bring 26 tests above score 80 on the Test Auditor (SPEC-055).
Root cause: tests were written without knowing the quality bar upfront.

We need an agent that knows the quality bar BEFORE writing the first line,
generates tests that score 80+ from the first attempt, and works across all
16 supported languages and all 14 test types.

## Solution

A new `test-architect` agent (Opus) that:

1. **Analyzes** input (spec, source code, or bug report)
2. **Plans** test strategy: which types, which language, which framework
3. **Generates** complete test files following 8 excellence patterns
4. **Self-verifies** BATS tests via test-auditor-engine.py

## 8 Excellence Patterns (Non-Negotiable)

From `feedback_test_excellence_patterns.md`:

| # | Pattern | Auditor Criterion |
|---|---------|------------------|
| 1 | setup()/teardown() with mktemp | C6: Isolation (10pts) |
| 2 | Safety verification (set -uo pipefail) | C2: Safety (10pts) |
| 3 | 3+ positive cases | C3: Positive (15pts) |
| 4 | 2+ negative cases | C4: Negative (15pts) |
| 5 | Edge cases (empty, boundary, zero) | C5: Edge (10pts) |
| 6 | Spec/doc reference | C8: Spec ref (10pts) |
| 7 | Diverse assertion types | C9: Assertions (10pts) |
| 8 | 60%+ coverage breadth | C7: Coverage (10pts) |

## 14 Test Types Supported

Unit, integration, E2E, validation (BATS), pipeline, regression, stress/load,
security, accessibility, visual regression, contract, mutation, property-based,
snapshot.

## 16 Language Packs

Each language maps to its native test framework. The architect selects the
correct framework based on project detection (language-packs.md).

| Language | Unit | Integration | E2E |
|----------|------|-------------|-----|
| C#/.NET | xUnit+FluentAssertions | TestContainers | Playwright |
| TypeScript | Jest/Vitest | Supertest | Playwright |
| Java | JUnit 5+Mockito | TestContainers | Selenium |
| Python | pytest | pytest+TestContainers | pytest-playwright |
| Go | testing+testify | dockertest | - |
| Rust | cargo test | - | - |
| PHP | PHPUnit | Laravel Dusk | - |
| Ruby | RSpec | Capybara | - |
| Bash | BATS | - | - |
| Others | See language-packs.md | - | - |

## Workflow

```
Input (spec | source | bug report)
  |
  v
1. ANALYZE — Detect language, identify test targets, classify test types needed
  |
  v
2. PLAN — Select frameworks, list test cases (positive/negative/edge), estimate
  |
  v
3. GENERATE — Write complete test files with all 8 excellence patterns
  |
  v
4. VERIFY — For BATS: run test-auditor-engine.py, fix if < 80
           For others: self-check against 8-pattern checklist
  |
  v
Output: test files + strategy summary
```

## Quality Gate

BATS tests generated by the architect MUST score >= 80 on the auditor.
Non-BATS tests MUST satisfy the 8-pattern checklist (manual verification).

## Non-Goals

- Replacing test-engineer (complementary: architect designs, engineer executes)
- Running tests (architect generates; test-runner executes)
- Modifying existing tests (use test-auditor feedback loop instead)
